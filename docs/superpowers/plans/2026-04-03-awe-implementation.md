# Awe Flutter App — Implementation Plan

**Date:** 2026-04-03  
**App:** Awe — cinematic sensory intervention that guides users from micro to cosmic scale to interrupt rumination.

---

## Architecture Summary

- **AudioEngine**: singleton `ChangeNotifier` owned by `AweApp`, injected via constructor
- **AppPreferences**: wraps `SharedPreferences` for `has_seen_intro`, montage resume state, audio position
- **Navigation**: `Navigator.pushReplacement` only — no routing library
- **Timed screens** accept optional `Duration` parameters as test seams
- **SensorService**: abstract class with `FakeSensorService` for gyro tests
- Audio wrapped in try/catch — tests verify state changes, not actual playback
- Placeholder assets generated by Python + Pillow (+ ffmpeg stubs)
